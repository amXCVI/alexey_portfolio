# Сайт-визитка Алексея С. — дизайн-решение

_Дата: 2026-06-18. Статус: утверждено, план в TAUSIK._

## Цель

Лёгкий, быстрый статический одностраничный сайт-визитка (онлайн-резюме)
Product-дизайнера. Источник истины — макет Figma, два фрейма:

- `1_screen` — `334:8926`
- `2_screen` — `387:9628`

Остальные фреймы в файле — черновики, **не верстаем**.

Файл: `figma.com/design/SagpfeC9cZBWcmybhRFqab` (страница `86:4`).

## Стек и инфраструктура

- **Astro** — статическая генерация, ~0 JS на клиенте. Компоненты `.astro`, TypeScript, обычный CSS (CSS-переменные), без тяжёлых зависимостей.
- Аккордеоны — нативный `<details>`/минимальный ванильный JS, без UI-фреймворка.
- **GitHub Pages** + GitHub Actions (авто-деплой при push в `main`). Учесть `base` путь.

## Визуальные токены (из Figma)

- Шрифт: **Manrope** (Google Fonts; Light / Regular / SemiBold). Бесплатный.
- Цвета: заголовок `#201f1f`, основной текст `#363636`. Фон светлый, карточки — светло-серые скруглённые. Кнопки — тёмные пилюли. Точная палитра/радиусы/отступы добираются из Figma при вёрстке.
- Типографика (пример hero): H1 44px SemiBold; подзаголовок 15px Light, line-height 32; интро 17px Regular, line-height 30.

## Структура страницы (компоненты)

1. **Hero** — фото + «Алексей С.» (H1) + подзаголовок (Product Designer, 11 лет) + вводный абзац.
2. **Навыки** — карточка: 6 пунктов списка + выделенная заметка (Figma / Agile).
3. **Tech Стек** — карточка: Figma; Adobe PS/AI; под-заголовок «Так же имею представления о:»; After Effects / Spine2D / Lottie.
4. **Опыт работы** — карточки проектов:
   - С превью-картинкой: *AR Spatially* (2022–2026), *Tap Topia* (2023–2026).
   - Широкие текстовые: *Sleogroup* (2019–2022) с тремя под-проектами; *NXmarketing* (2016–2019).
   - У каждой — кнопка-аккордеон «Кейс продукта» / «Моя роль и достижения».

## Утверждённые решения

- **Аккордеоны:** механика рабочая; раскрытый контент кейсов в этих фреймах не нарисован (он в черновиках) → внутри аккуратная заглушка («Контент кейса скоро») с готовой структурой под будущий контент.
- **Контакты/футер:** в макете нет → не добавляем.
- **Адаптив:** мобильного макета нет → responsive на наше усмотрение (одна колонка, уменьшение шрифтов/отступов) на основе десктопа.
- **Шрифты/картинки:** точные токены и ассеты (фото, превью проектов, иконки) экспортируются из Figma при вёрстке.

## Нефункциональные требования

- Производительность: Lighthouse Performance ≥ 95, минимум клиентского JS.
- Доступность: семантическая разметка, корректные `alt`, доступные аккордеоны (`aria-expanded`, клавиатура), достаточный контраст.
- SEO: `<title>`, meta description, Open Graph, `lang="ru"`.

## Вне объёма (сейчас)

Страницы/контент кейсов, блок контактов, мультиязычность, CMS, аналитика, остальные (черновые) фреймы макета.
